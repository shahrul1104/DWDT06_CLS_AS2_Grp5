Introduction

Because of the high level of market fragmentation in Asia, doing business is particularly difficult. Organizations require infrastructure that can accommodate better business continuity, increased collaboration, and higher levels of automation without drastically increasing capital expenditures in order to better maintain services, develop new products, and run regional operations on a day-to-day basis. In this report the author choose Amazon Web Services as cloud migration service because

  • AWS is cost-effective.
  • Availability and High performance.
  • 24/7 support services.

Is my organization ready for the cloud?

Migration Readiness Assessment is a process where an organization examine their application and data in order to determine whether it is ready to be moved to the cloud. It helps to reduce risks, cost, necessary process, potential migration paths and post-migration benefits.

The scope and business cases for migration

An organization should determine why and to what extent it needs to migrate to the cloud.

Evaluate Available In-House Resources

An organization may need to assess its own resources before embarking on a cloud migration. IT professionals who aren't familiar with the process should study how to do it and talk to peers. This can significantly increase the cost of cloud migration, so firms might need to hire an expert.

Estimate timeline and budgets

An organization will obtain more accurate timeframe and budget estimate before the project starts if an organization use AWS Professional Services

Prioritize what application to move first

It is uncommon for an organization to shift everything to the cloud at the same time. Determine which applications or workloads to migrate first by doing a strategic portfolio study.

Importance of security

The consumer and the cloud service provider share responsibilities for cloud security. Work with your cloud provider to develop comprehensive and robust security rules.

Choosing the right Cloud Provider Service

Reliability

Paper offers in-depth best-practice guidelines for creating dependable workloads on Amazon Web Services. The Reliability pillar considers a workload's ability to perform its intended purpose correctly and consistently. This includes the ability to run the workload and test it during its entire lifecycle.

Certification and Standards

Best practices and standards are demonstrated by providers who adhere to recognized standards and quality frameworks. If security is a concern, search for providers who have received certifications such as ISO 27001 or the government's Cyber Essentials Scheme.

Migration Support, Vendor Lock in & Exit Planning

Be wary of "enhancement creep", where service providers modify configurations, policies, technologies etc., and in doing so introduce lock-in factors as part of your service. It's worth learning about a CSP's practices before signing a contract because leaving their service isn't usually a quick or pleasant transfer.

cost

Cloud migration is neither inexpensive nor simple, picking the proper plan is an unavoidable decision. All cloud service providers must be compared based on the various price plans and licenses. It's a good idea to set aside a budget for cloud infrastructure setup and keep to the pre-determined comparison criteria.

Information Security

Cloud providers should ensure their data security measures are risk-based and clearly support an organization's security policies and procedures. Make sure that all user access and activity can be audited, and that security roles and responsibilities are clearly defined in the contacts or business policies documents.

How can we start the cloud migration project? What are the phases?



Phase 1: Migration Preparation

The first phase of migration will involve in developing clear structure of the organization current operation, goals and expectation for migration. Expectation’s migration, in addition, need to be realistic and support the business goals. It needs to be realistic in terms of budget, feasibility of migration and shareholder perspective

Phase 2 – Portfolio discovery and Planning

Once the organization has chosen the right migration, the next step will be creating a clear migration plan in terms of strategies and priorities. To do this, the organization nee to create an inventory of the current infrastructure, configuration and assets. Firmly determine on how all resources are being used and what requirement exist and what is needed to reported.

Phase 3 and 4 – Designing, Migrating and Validating Application

Phase 3 and 4 are the stage where an organization will start shifting and lifting process with providing excellent tetsing ground for businesses plan.

Phase 5 – Operation

As application are migrated, businesses need to optimized new foundation [13]. To accomplish this, the organization need to endure any workload to cloud resources and begin to turn-off on-premise operations. After migration, the organization also need to monitor the resources and workloads. How can we ensure successful migration?

Step 1

Planning & assessment

Planning and assessment should include what the organization would expect during the migration process. The organization should appoint someone as the migration architect that is responsible for making sure the migration is successful. This person should be able to ensure all aspects of the migration are completed on time and within budget.

Step 2

Shallow cloud integration or Deep cloud integration

Shallow cloud integration is where the on-premise processes are moved to the cloud without or little changes to the server. Amazon Web Services provides features such as AWS Lambda which allows organization to use auto-scaling and dynamic load balancing to ease the process.

Step 3

Ensuring a working and moving momentum

To create a working momentum, the migration architecture should do a small migration task that the organization can do successfully and without hassle to create a pushing drive and to ensure the organization will have proper safeguards in place to keep the migration momentum going, even when there are risks of slippage, such as when your staff takes time off or when tools don’t work as designed for specific workloads.

What are the pitfalls to be avoided?

Question 5

  • Not Planning thoroughly or not enough migration strategy created

    Migrating to the cloud without a plan in place wastes time and money for the company. This will result in unanticipated time and cost requirements. Most significantly, switching to the Cloud frequently results in security issues. Protecting the integrity of data while it is transported should be a top priority.


  • Underestimating cost

    Cost savings mean companies don't have to deal with expensive data centers, gear, or keep applications running 24/7 via the internet.


  • Migrating all data at once

    Companies should never move all of their data at once. Depending on the amount of data to be transmitted, cloud migrations might take months or even years. When enterprises move data all at once, they frequently fail to develop a "proof-of-concept" or minimal viable cloud.


  • Not considering managed services

    Professional services providers can help an organization with all elements of the process. Consider whether a managed services partner would be a good fit for the company's needs and budget.


What are the post operations after migration?

Question 6

  • Cloud Monitoring

    While the cloud provider monitors systems for basic health and availability, enterprises must do their own monitoring. This monitoring may include monitoring virtual machines, operating system resources, and application availability.


  • Cloud Computing Training

    An organization's IT staff may need training to ensure they understand the roles they will continue to play in the cloud. Development teams must grasp how cloud computing affects how they design, construct, test, and deploy apps.


  • Closure

    It is critical to do a clean-up immediately upon migration, including the shutdown and removal of any redundant systems, says former Google COO Larry Page. Such systems and links should be decommissioned in a controlled but prioritized manner, he says.


  • Evaluation

    Once the transfer is complete, assess whether the migration was successful and accomplished everything that was set out to do. Such review and documentation is essential both before and after the migration. It keeps the team honest about their aims and any decisions taken. At the same time, it makes success apparent.

What are the concerns and issues encounter by others?

Question 8

  • Downtime

    The migration process may necessitate taking in-house servers offline for a period of time. However, if effective backup and resource allocation are not in place, outages can be terrible for application performance and, by extension, client loyalty.


  • Cost

    Many firms migrate to the cloud without adequately assessing the workload, resulting in erroneous cost estimates, furthermore, because cloud environment is dynamic, expenses can rise when enterprises decide to add new services or scale up existing one


  • Compliance and Data Security

    Cloud environments can be as secure as on-premise environments, but there is a risk of a "security vacuum". Companies want to know what data is secured in the cloud and where it is stored. AWS provides a safe cloud infrastructure that is consistent with industry standards.


  • Lacking strategy

    Managers often approach employees after becoming stuck after investing in a significant amount of effort on migration. The most common error that can stop an organixation from truly knowing the benefits of the cloud is missing to have a defined business goal or a well-thought-out migration plan, say experts.
